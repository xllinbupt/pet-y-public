# Pet Y

Pet Y is an early concept for agent-generated social desktop pets.

The project explores a desktop pet that is created and customized through an Agent, lives locally on the user's desktop, and can truly visit friends' desktops through a lightweight relay and open protocol.

## MVP Prototype

The first runnable MVP now has a native macOS desktop Runtime plus a no-dependency Node Relay.

It includes:

- A lightweight in-memory Pet Relay.
- Two sample users: Alice and Bob.
- Two native macOS Runtime processes.
- Transparent floating pet windows on the desktop.
- A lightweight menu bar control for each Runtime.
- Pet profile registration.
- A real visit session flow.
- Visitor projection on the host desktop.
- Click, drag, feed, and return interactions.
- Interaction event relay.
- Memory receipt and life log generation.
- Pet profiles loaded from local Pet Life Packs.
- Sprite sheet animation for local pets.
- Quick actions under the pet, including petting, sleeping, and throwing a ball.
- A fetch-ball interaction where Momo runs away while shrinking, picks up the ball, then carries it back while growing to normal size.

The included Momo and Yuzu pets are sample life packs, not the final product. The product is the Skill-driven creation flow that generates life packs for Runtime.

## Run

Start the Relay:

```bash
npm start
```

To expose the Relay from a Linux server, start it on all interfaces:

```bash
npm run start:public
```

In two other terminals, start Alice and Bob desktop Runtimes:

```bash
./scripts/run-desktop-alice.sh
./scripts/run-desktop-bob.sh
```

For a normal local user, start Runtime with the generic launcher. It creates a local identity automatically on first launch:

```bash
./scripts/install-runtime.sh
./scripts/run-desktop.sh
```

Normal users do not need Xcode or local Swift compilation. The launcher uses the downloaded native Runtime in `bin/PetYDesktop`. Developers changing the Swift Runtime can still build it directly:

```bash
npm run build:desktop
```

Run a custom generated life pack:

```bash
PET_Y_LIFE_PACK=life-packs/luma/pet-life.json ./scripts/run-desktop.sh
```

Run against the shared Aliyun Relay:

```bash
PET_Y_RELAY=http://47.99.98.43:8787 PET_Y_LIFE_PACK=life-packs/luma/pet-life.json ./scripts/run-desktop.sh
```

## Demo Flow

1. Start the Relay.
2. Start Alice and Bob desktop Runtimes.
3. Alice and Bob automatically register their pet profiles.
4. From Alice's Pet Y menu bar item, choose "让宠物去 bob 串门".
5. Momo disappears from Alice's desktop and leaves an away sign.
6. Momo appears as a floating visitor pet on Bob's desktop.
7. In Bob, click Momo to open quick actions, drag it, or choose a simple interaction.
8. From Bob's Pet Y menu bar item, choose "请回来访宠物".
9. Alice receives a memory receipt and life log entry, and Momo returns home.

Bob's desktop Runtime must be running for Momo to visit. If Bob is offline, Relay rejects the visit instead of creating a fake trip.

If Bob's Runtime quits during a visit, Relay automatically ends the visit after the host heartbeat expires. Momo returns to Alice with a short note instead of staying lost.

## Friends

The MVP now has a lightweight invite flow instead of a full account system.

From the `Pet Y` menu bar item:

- Choose `邀请好友一起玩` to copy a complete share message with setup instructions, Relay URL, and your invite code.
- Give that message to another Runtime user.
- The other user gives the message to Codex. Codex should install and prepare Pet Y, interview them, generate their own pet, then connect to the same Relay.
- The other user chooses `输入邀请码加好友` and pastes your invite code.

For debugging, `复制我的邀请码` still copies only the raw invite code.

Relay then creates a two-way friendship. Friend online status is refreshed through the Runtime heartbeat, and the pet-side `串门` action shows whether each friend is at home.

This is still a local prototype identity model. It is not yet a production account system with login, recovery, or multi-device sync.

## Real Friend MVP

The current target flow is:

1. Install the project-local Pet Y Skill.
2. Start a shared Relay.
3. Each friend uses Codex with this project and the Pet Y Skill to create their own Pet Life Pack through an interview.
4. Each friend runs their own desktop Runtime against the same Relay.
5. Friends use `邀请好友一起玩` from the menu bar item, or the pet-side `邀请` quick action.
6. After both pets are online, click the pet, choose `串门`, and select the online friend.

Install the Skill locally:

```bash
./scripts/install-pet-y-skill.sh
```

Restart Codex after installing it.

Shared Relay candidate:

```text
http://47.99.98.43:8787
```

On the Aliyun ECS machine, the Relay should be launched from this project with:

```bash
HOST=0.0.0.0 PORT=8787 npm start
```

The ECS security group must allow inbound TCP `8787`. The MVP Relay is still HTTP and in-memory; it is enough for a live prototype, but not a production account service.

A friend can create a pet with:

```bash
npm run create:pet -- --name Luma --pet-id pet_luma --style pixel --form "会发光的小星星" --signature glow --personality "安静、贴心，会在你分心时轻轻闪一下"
```

Then install the native Runtime and run it:

```bash
./scripts/install-runtime.sh
PET_Y_RELAY=http://47.99.98.43:8787 PET_Y_LIFE_PACK=life-packs/luma/pet-life.json ./scripts/run-desktop.sh
```

Runtime creates a stable local identity at:

```text
~/Library/Application Support/PetY/identity.json
```

Pet memories and life logs remain local on the owner's machine.

## Current Limits

The native Runtime is currently macOS-only and uses a simple Swift/AppKit implementation. It is distributed as a prebuilt command-line binary for normal users, not a `.app` bundle yet.

The desktop Runtime no longer opens a large control window by default. Runtime controls live in the macOS menu bar under `Pet Y`.

The Relay is in-memory. Restarting the server clears sessions and profiles.

The desktop Runtime persists each pet's local state on the owner's machine. Life logs and memory receipts are stored under:

```text
~/Library/Application Support/PetY/<user-id>/pet-state.json
```

Relay data can disappear in this MVP, but the owner's local pet memories survive Runtime restarts.

## Pet Life Packs

Runtime loads sample pets from:

```text
life-packs/alice-momo/pet-life.json
life-packs/bob-yuzu/pet-life.json
```

Pet Skill should eventually generate these life packs through an interview flow instead of hand-writing Momo/Yuzu.

The project now includes a first deterministic generator:

```bash
npm run create:pet -- --name Luma --pet-id pet_luma --owner alice --style pixel --form "会发光的小星星" --signature glow --personality "安静、贴心，会在你分心时轻轻闪一下"
```

This creates a new Pet Life Pack under:

```text
life-packs/luma/
```

It also creates a model-generation task list:

```text
life-packs/luma/asset-generation-tasks.md
```

Pet art should be generated with an image model or curated by a human. The Skill should not draw pets with code except for disposable engineering tests.

Reusable Skill presets live under:

```text
skill-presets/styles/
```

The first reusable example is a pixel-style dog:

```text
skill-presets/styles/pixel/examples/pixel-dog/
```

It includes sprite sheet animation state specs for running, sitting, sleeping, and carrying a ball.

The Skill should generate both the pet's identity and its action states. A dog might run and fetch a ball; a small star might float, glow, and dim for sleep; a book might flap pages or close itself. Good generated examples should be saved back into `skill-presets/styles/<style>/examples/` so future pets in the same style stay visually consistent.

New life packs should prefer generic Runtime action names: `idle`, `move`, `rest`, `sleep`, and `signature_xxx`. The native Runtime resolves these intent-level actions and still supports older Momo-style states like `run`, `sit`, and `carry_ball` as fallbacks.

The current Runtime plays `animation_states.idle` when the life pack has an `assets/idle.png` sprite sheet. If the asset is missing, it falls back to the built-in drawn pet.

Momo currently uses these local animation triggers:

- Idle: default loop.
- Interaction menu: click Momo to open quick actions.
- Pet: choose "摸摸" from the quick actions.
- Run: drag Momo and release.
- Sleep: choose "睡觉", or leave Momo idle for a short while.
- Carry ball: Momo returns home after a visit.
- Fetch ball: choose "丢球"; Momo runs away while shrinking, picks up the ball, then carries it back while growing to normal size.

Placeholder sprite sheets can be regenerated with:

```bash
node scripts/generate-placeholder-sprites.js
```

Generated chroma-key sprite sheets can be normalized into Runtime-ready `256x64` transparent PNG files with:

```bash
python3 scripts/prepare-sprite-sheet.py --input <generated.png> --output <life-pack>/assets/idle.png --frames 4 --frame-size 64
```
