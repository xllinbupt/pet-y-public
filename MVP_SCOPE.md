# MVP Scope

## 1. MVP 目标

第一版要验证的不是“完整宠物生态”，而是一个最小但真实的闭环：

> 用户通过 Agent 生成一只桌面宠物，让它在本地桌面生活，并能主动派它去在线好友桌面真实串门。好友能看到并互动，宠物回家后能把经历变成生活日志。

MVP 必须证明三件事：

- Agent 能生成一只可运行、可定制的桌面宠物。
- 宠物能以受控投影的方式真实出现在好友桌面。
- 互动事件能回到主人本地，并变成宠物自己的经历。

## 2. MVP 用户流程

### 2.1 创建宠物

用户在 Codex、Claude 等 Agent 客户端中启动 Pet Skill。

Agent 通过简短访谈生成：

- 宠物名称
- 风格
- 形态
- 性格
- 基础动作
- 基础互动
- 本地 Runtime 项目
- 初始 `PetProfile`

第一版访谈不要过长。目标是让用户尽快看到宠物活起来。

### 2.2 本地生活

宠物在用户桌面上出现，并具备基础桌面行为：

- 待机
- 走动
- 睡觉
- 坐下或叼东西等动作状态
- 被点击时反应
- 被拖拽时跟随鼠标
- 通过宠物下方快捷动作触发互动
- 基础气泡表达

宠物有本地状态和生活日志，但第一版不要求复杂长期记忆。

### 2.3 建立好友

用户能添加好友，并允许好友宠物自动来访。

第一版好友机制可以很轻：

- 用用户 ID 或邀请链接添加好友
- 双方确认后成为好友
- 好友默认允许自动来访
- 用户可以禁访或解除好友

### 2.4 发布宠物名片

本地 Runtime 将轻量 `PetProfile` 发布到 Relay。

Relay 保存的是快照，不是宠物真身。

这一步也可以理解为宠物注册。只有发布过 `PetProfile` 的宠物，才可以发起真实串门。因为 Relay 和好友 Runtime 需要用这份名片确认宠物身份、主人归属、素材版本、协议版本和可投影能力。

### 2.5 主动派出串门

用户选择一个在线好友，点击“让宠物去串门”。

系统创建 `VisitSession`。

如果好友在线且允许自动来访，好友 Runtime 创建来访宠物投影。

### 2.6 好友互动

好友在桌面上真实看到来访宠物。

好友可以：

- 点击它
- 拖拽它
- 给它一个简单虚拟食物或礼物
- 请它回家

第一版不要求复杂聊天，也不要求来访宠物主动读取任何宿主信息。

### 2.7 宠物回家

串门结束后，互动事件回传给主人 Runtime。

主人 Runtime 根据事件生成：

- 一条生活日志
- 一句宠物自己的表达
- 简单关系牵连记录

示例：

> Momo 去了小林的桌面。小林把它拖到右上角，它在那里待了一会儿，还收到了一颗草莓。

## 3. MVP 必须包含

### 3.1 Pet Skill 最小能力

- 引导用户生成宠物设定
- 支持至少 3 种风格预设：像素风、手绘贴纸风、仿真毛绒 / 真实风
- 生成本地项目结构
- 生成初始宠物配置
- 生成或引用基础素材
- 生成 `PetProfile`
- 提供继续定制宠物的说明

### 3.2 Pet Runtime 最小能力

- 桌面端透明窗口
- 显示本地宠物
- 显示来访宠物投影
- 基础移动和待机动画
- 点击互动
- 拖拽互动
- 简单气泡文本
- 本地生活日志
- 与 Relay 建立连接
- 创建和接收 `VisitSession`
- 记录并上传 `InteractionEvent`

当前 MVP 使用 macOS Swift/AppKit 实现原生桌面 Runtime。浏览器页面只作为协议调试 playground，不算主要 Runtime。

当前样例 Momo 已经具备像素风 sprite sheet 动画，包括待机、跑动、坐下、睡觉和叼球。丢球互动会把屏幕移动和绘制缩放组合起来，表现“跑远变小、叼球回来变大”。

### 3.3 Pet Relay 最小能力

- 用户注册或本地身份绑定
- 设备在线状态
- 好友关系
- 自动来访权限
- 保存 `PetProfile` 快照
- 创建 `VisitSession`
- 路由来访请求
- 接收并转发互动事件
- 基础禁访 / 请回能力

### 3.4 Pet Protocol 最小对象

第一版只需要稳定以下对象：

- `PetProfile`
- `HostRules`
- `VisitSession`
- `PetProjection`
- `InteractionEvent`
- `MemoryReceipt`

### 3.5 安全边界

MVP 必须遵守：

- 来访宠物不能执行远程代码。
- 来访宠物不能读取宿主文件。
- 来访宠物不能读取宿主屏幕。
- 来访宠物不能操作宿主鼠标键盘。
- 宿主 Runtime 对来访宠物拥有最终控制权。
- 宿主可以请回、静音、禁访来访宠物。

## 4. MVP 暂不包含

以下内容先不进入第一版：

- 宠物主动决定出门
- 多只来访宠物同时复杂互动
- 宠物之间复杂关系图谱可视化
- 长期后台生活模拟
- 合盖后真实后台串门
- 完整 AI 对话
- 主人之间的即时通讯
- 礼物经济系统
- 复杂物品背包
- 公开宠物市场
- 跨平台安装包发行
- P2P 直连
- 复杂权限策略
- 多设备同步同一只宠物

这些不是否定方向，只是为了保证第一版能闭环。

## 5. 第一版技术形态建议

技术选型还可以继续讨论，但 MVP 可以先按以下形态理解：

- Pet Skill：一套 Agent 使用的生成说明、模板和脚手架。
- Pet Runtime：本地桌面应用，负责透明窗口、宠物渲染、互动和本地状态。
- Pet Relay：轻量 Web 服务，负责在线状态、好友、profile、visit session 和 event relay。
- Pet Protocol：JSON 对象和事件约定。

MVP 不要求服务器运行宠物实例。宠物完整状态保存在主人本地。

## 6. MVP 验收标准

第一版完成时，应该可以演示以下流程：

1. 用户 A 通过 Agent 生成一只宠物 Momo。
2. Momo 出现在用户 A 的桌面上。
3. 用户 A 点击或拖拽 Momo，Momo 有基础反应。
4. 用户 B 也运行 Runtime，并和用户 A 成为好友。
5. 用户 A 主动派 Momo 去用户 B 桌面串门。
6. 用户 B 的桌面上真实出现 Momo 的来访投影。
7. 用户 B 点击、拖拽或投喂 Momo。
8. Momo 回到用户 A 桌面。
9. 用户 A 看到 Momo 带回来的生活日志。
10. 系统记录 Momo 和用户 B 本地宠物之间的简单互动牵连。

只要这个闭环成立，MVP 就成功。

当前原型已经跑通这个闭环的核心版本：Alice/Bob 两个桌面 Runtime 连接 Relay，Momo 可以主动串门、离家留牌、在 Bob 桌面出现、记录互动、在 Bob Runtime 掉线后自动回家，并把记忆保存在 Alice 本地。

## 7. 后续版本方向

MVP 后可以逐步扩展：

- 宠物主动串门
- 宠物之间自动邀约
- 更丰富的人宠互动
- 更完整的宠物关系图谱
- 宠物生活日志 UI
- 礼物和纪念物
- 离线生活模拟
- 合盖或休眠后的生活补全
- Agent 驱动的宠物二次改造
- 更开放的第三方宠物协议
- 公共 Relay 或自托管 Relay
